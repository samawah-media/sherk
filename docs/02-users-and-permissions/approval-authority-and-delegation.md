# Approval Authority and Delegation: شريك

**المرحلة:** Phase 03 - Roles, Permissions, Visibility & Delegation Model  
**نوع الوثيقة:** Approval Authority and Delegation  
**الحالة:** Draft for owner review  
**آخر تحديث:** 2026-06-22  

## 1. الغرض

هذه الوثيقة تحدد من يراجع، من يعمد داخليا، من يرسل للعميل، من يعتمد باسم العميل، وكيف يعمل التفويض المؤقت. لا تحسم الأسئلة المفتوحة غير المعتمدة تلقائيا.

## 2. مسار السلطة المختصر

```text
تنفيذ ورفع نسخة
→ طلب مراجعة داخلية
→ تعديل داخلي أو تعميد داخلي
→ إرسال للعميل إذا كان مطلوبا
→ اعتماد عميل أو طلب تعديل
→ تسليم نهائي وإغلاق
```

| القرار | صاحب السلطة الافتراضي | Audit Event | التصنيف |
| --- | --- | --- | --- |
| طلب مراجعة داخلية | Owner أو عضو تنفيذ مخول | `submitted_for_internal_review` | Confirmed |
| طلب تعديل داخلي | PM/MM/QR مفوض | `internal_change_requested` | Confirmed |
| التعميد الداخلي | PM/MM/QR مفوض | `internal_approval_granted` | Confirmed |
| إرسال للعميل | PM/MM أو AM مخول بعد التعميد | `deliverable_sent_to_client` | Confirmed |
| اعتماد العميل | Client Approver | `client_approval_granted` | Confirmed |
| طلب تعديل العميل | Client Approver | `client_change_requested` | Confirmed |
| التسليم النهائي | PM/MM/Executive أو AM مخول | `deliverable_delivered` | Confirmed |
| Emergency Override | Executive / Tenant Owner | `emergency_override_used` | Assumed |

### 3.4 متى يلزم مراجع آخر؟

يلزم مراجع آخر عندما:

- صاحب التعميد هو owner أو آخر رافع للنسخة.
- المخرج عالي الحساسية أو مرتبط بحملة مهمة.
- يوجد تعديل SLA أو تجاوز باقة مرتبط بالمخرج.
- حصل اعتراض عميل سابق.

| التصنيف | التوصية |
| --- | --- |
| Assumed | V1 يبدأ بمراجع واحد مستقل عن المنفذ عند الإمكان. |
| Open Question | هل توجد مخرجات تحتاج اعتمادا داخليا متعدد المستويات؟ |

### 3.8 كيف يعمل التفويض المؤقت؟

التفويض المؤقت:

- له بداية ونهاية.
- لا يتجاوز صلاحيات المفوض الأصلي.
- يحدد Client/Deliverable/Approval type.
- يسجل سبب التفويض.
- ينتهي تلقائيا أو يلغى مبكرا.
- لا يمحو مسؤولية الفاعل الذي نفذ القرار.

| التصنيف | التوصية |
| --- | --- |
| Confirmed | كل تفويض يحتاج Audit Event. |
| Assumed | التفويض في V1 يدوي بإدارة Tenant وليس self-service كامل. |

| تفويض مؤقت | عملي | يحتاج تدقيق | التوصية |
| Emergency Override | يحل الطوارئ | خطر الثقة | فقط للحالات الحرجة |

### 3.10 كيف يسحب الاعتماد؟

| نوع الاعتماد | هل يسحب؟ | القاعدة | التصنيف |
| --- | --- | --- | --- |
| تعميد داخلي قبل الإرسال | نعم | سبب وسجل `internal_approval_revoked` | Assumed |
| تعميد داخلي بعد الإرسال للعميل | Conditional | يفتح مسار تصحيح أو سحب إرسال | Open Question |
| اعتماد عميل قبل التسليم | Conditional | يحتاج قرار إداري وسبب | Open Question |
| بعد التسليم النهائي | لا يسحب عادة؛ يستخدم إعادة فتح/تصحيح | Confirmed |

### 3.11 متى يستخدم Emergency Override؟

يستخدم فقط عندما:

- هناك خطر عميل أو موعد حرج.
- لا يوجد معتمد متاح.
- القرار لا يمكن انتظاره.
- الإدارة العليا تتحمل المسؤولية.
- يسجل السبب وتتم مراجعة لاحقة.

لا يستخدم لتجاوز التعميد الداخلي كمسار طبيعي.
