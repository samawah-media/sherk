## 1. الغرض

هذا الملف يحول الأطراف الثلاثة إلى "وظائف" يريدون إنجازها في لحظات محددة. الهدف هو منع تحويل المنتج إلى قائمة ميزات، وربط كل ميزة في V1 بسبب استخدام واضح.

| التصنيف | المعلومة |
| --- | --- |
| Confirmed | المنتج يخدم العميل، فريق العمل، والإدارة. |
| Assumed | JTBD أدناه فرضيات تشغيلية تحتاج تحقق عبر مقابلات. |
| Open Question | ما الوظائف الأعلى ألما لدى كل طرف في التشغيل الحالي؟ |

## 2. العميل: معرفة ما ينتظر قراره

### When

عندما يدخل العميل لمتابعة العمل أو يصله إشعار أن مخرجا ينتظر موافقته.

### I want to

أرى النسخة المعتمدة داخليا فقط، وأقرر اعتمادها أو طلب تعديل عليها.

### So I can

أحافظ على سرعة التسليم دون أن أغرق في تفاصيل العمل الداخلي أو النسخ غير النهائية.

### Functional Jobs

- فتح قائمة "بانتظار موافقتي".
- مراجعة معاينة أو ملف المخرج.
- كتابة ملاحظة تعديل واضحة.
- اعتماد النسخة.
- الرجوع لسجل قراراته.

### Emotional Jobs

- الشعور بالسيطرة على العمل دون متابعة مستمرة.
- الثقة أن النسخة المعروضة جاهزة وليست مسودة داخلية.
- الاطمئنان أن موافقته موثقة.

### Social Jobs

- الظهور داخليا كمدير منظم يعرف حالة المخرجات.
- تجنب لوم التأخير عندما يكون القرار عنده.

### Current Alternatives

- واتساب لروابط المعاينة والتعليقات.
- Drive للملفات.
- رسائل بريد للتأكيد.

| التصنيف | المعلومة |
| --- | --- |
| Confirmed | العميل يعتمد أو يطلب تعديل بعد التعميد الداخلي. |
| Confirmed | قرار العميل يجب أن يسجل في Audit Log. |
| Assumed | العميل يفضل أقل عدد شاشات ممكن. |
| Open Question | هل يحتاج العميل اعتمادا من الهاتف غالبا أم سطح المكتب؟ |

## 3. العميل: متابعة العقد والمتبقي

### When

عندما يريد العميل معرفة ما تم إنجازه وما تبقى من الباقة أو العقد.

### I want to

أرى ملخصا واضحا للمخرجات المتفق عليها والمنجزة والمتبقية.

### So I can

أعرف قيمة العمل الجاري وأخطط للموافقات القادمة دون سؤال مدير الحساب.

### Functional Jobs

- رؤية إجمالي المخرجات المتفق عليها.
- رؤية المنجز والمتبقي.
- رؤية المخرجات الجارية.
- الوصول للملفات النهائية.

### Emotional Jobs

- تقليل القلق من أن العمل متأخر أو غير واضح.
- الشعور بأن الوكالة شفافة ومنظمة.

### Current Alternatives

- تقرير يدوي من مدير الحساب.
- ملف Excel أو عرض شهري.

| التصنيف | المعلومة |
| --- | --- |
| Confirmed | لوحة العميل تعرض المتبقي والتقدم والملفات. |
| Assumed | العميل لا يحتاج تفاصيل مالية متقدمة في V1. |
| Open Question | هل يريد العميل رؤية قيمة مالية أم عدد المخرجات فقط؟ |

## 4. فريق العمل: تنفيذ المخرج الصحيح

### When

عندما يبدأ عضو الفريق يومه أو يستلم مخرجا جديدا.

### I want to

أرى المخرجات المسندة لي، ملفاتها، تعليقات العمل، وحالة SLA.

### So I can

أنفذ النسخة الصحيحة وأرفع العمل للمراجعة دون البحث في رسائل وملفات متفرقة.

### Functional Jobs

- رؤية "مهامي".
- فتح كارت المخرج.
- قراءة الوصف والتعليقات الداخلية.
- تنزيل الملفات اللازمة.
- رفع نسخة جديدة.
- إرسال للمراجعة الداخلية.

### Emotional Jobs

- تقليل التوتر من ضياع ملاحظة أو ملف.
- الشعور بأن المسؤوليات واضحة.
- تجنب اللوم على تأخير سببه العميل أو الإدارة.

### Social Jobs

- إظهار التزامه بالجودة والوقت.
- إثبات ما تم رفعه ومتى.

### Current Alternatives

- واتساب للتوجيهات.
- Drive للملفات.
- Trello/Excel للمتابعة.

| التصنيف | المعلومة |
| --- | --- |
| Confirmed | الفريق يرى مهامه ولوحة Kanban الداخلية. |
| Confirmed | الفريق لا يرسل للعميل مباشرة افتراضيا. |
| Assumed | أكبر فائدة للفريق هي توحيد سياق المخرج. |
| Open Question | هل يحتاج الفريق مهاما فرعية إلزامية في V1 أم تكفي حالة المخرج؟ |

## 5. الإدارة: التعميد الداخلي قبل العميل

### When

عندما يرفع الفريق نسخة ويحول المخرج إلى "جاهز للمراجعة الداخلية".

### I want to

أراجع النسخة، أرى Checklist الجودة، وأقرر اعتمادها داخليا أو طلب تعديل.

### So I can

أحمي جودة العمل ولا يرى العميل إلا نسخة مناسبة ومعتمدة.

### Functional Jobs

- رؤية قائمة بانتظار مراجعة داخلية.
- فتح تفاصيل المخرج.
- مراجعة الملفات والنسخ والتعليقات.
- طلب تعديل داخلي.
- اعتماد داخلي.
- تسجيل سبب القرار.

### Emotional Jobs

- تقليل خطر الإحراج أمام العميل.
- الشعور بأن الجودة تحت السيطرة.

### Social Jobs

- الحفاظ على صورة الوكالة أمام العملاء.
- إثبات الالتزام بإجراءات جودة واضحة.

### Current Alternatives

- مراجعة عبر واتساب أو Drive.
- ملاحظات شفوية أو رسائل متفرقة.

| التصنيف | المعلومة |
| --- | --- |
| Confirmed | التعميد الداخلي بوابة قبل ظهور العمل للعميل. |
| Confirmed | طلب تعديل داخلي لا يظهر للعميل. |
| Assumed | Checklist الجودة تكون مرنة حسب نوع المخرج لاحقا. |
| Open Question | ما الحد الأدنى من Checklist في MVP لكل نوع مخرج؟ |

## 6. الإدارة: معرفة مالك التأخير

### When

عندما يتأخر مخرج أو يقترب من موعد التسليم.

### I want to

أعرف هل التأخير عند الفريق، الإدارة، العميل، أو بسبب ملف ناقص.

### So I can

أتدخل في المكان الصحيح وأقدم تقريرا عادلا عن SLA.

### Functional Jobs

- رؤية SLA status.
- رؤية delay owner.
- رؤية وقت انتظار العميل.
- رؤية المخرجات at risk وoverdue.
- تصدير أو عرض ملخص بسيط.

### Emotional Jobs

- تقليل الغموض واللوم غير العادل.
- الشعور أن الإدارة تملك زمام التشغيل.

### Social Jobs

- تقديم صورة احترافية للعميل عن سبب التأخير.
- حماية الفريق من احتساب انتظار العميل عليه.

| التصنيف | المعلومة |
| --- | --- |
| Confirmed | SLA يتوقف عند انتظار العميل ولا يحسب على سماوة. |
| Confirmed | سبب التوقف والاستئناف يسجل في Audit Log. |
| Assumed | delay owner سيحتاج لغة مبسطة في المنتج. |
| Open Question | هل يعرض delay owner للعميل أم للإدارة فقط؟ |

## 7. الإدارة: إدارة نطاق العقد والباقة

### When

عندما يتم الاتفاق على عقد أو باقة أو تحديث مخرجات.

### I want to

أدخل المخرجات المتفق عليها وأربطها بالعميل والعقد أو الباقة.

### So I can

أحسب المنجز والمتبقي وأمنع العمل خارج الاتفاق دون وضوح.

### Functional Jobs

- إنشاء عميل.
- إنشاء عقد أو باقة.
- إضافة مخرجات.
- تحديد أنواع وكميات.
- ربط مخرج ببند عقد أو باقة.
- متابعة الاستهلاك.

### Emotional Jobs

- الثقة أن كل عمل محسوب.
- تقليل النزاع حول "هل هذا داخل الباقة؟".

| التصنيف | المعلومة |
| --- | --- |
| Confirmed | العقود والباقات جزء من قلب V1. |
| Assumed | تفاصيل الفوترة المتقدمة خارج V1، لكن تتبع الباقة داخل V1. |
| Open Question | هل توجد باقات كمية ثابتة لكل نوع مخرج أم عقود مرنة بالكامل؟ |

## 8. وظائف مشتركة

### حفظ سجل النشاط

- عندما يتغير مخرج أو يضاف تعليق أو يعتمد طرف ما، يجب تسجيل الحدث.
- السبب: التوثيق، المساءلة، وحماية العلاقة مع العميل.

### حماية الخصوصية

- عندما يفتح أي مستخدم بيانات، يجب أن يرى ما يخص نطاقه فقط.
- السبب: شريك منتج متعدد العملاء، وعزل البيانات شرط ثقة أساسي.

### إدارة الملفات

- عندما يرفع أحد ملفا، يجب ربطه بسياق: عميل، مخرج، عقد، رؤية.
- السبب: الملفات ليست مساحة عامة، بل جزء من دورة التسليم.

## 9. Prioritized Jobs for MVP

1. العميل يعتمد أو يطلب تعديلا على مخرج معتمد داخليا.
2. الفريق ينفذ ويرفع نسخة ويرسلها للمراجعة الداخلية.
3. الإدارة تعمد داخليا وترسل للعميل.
4. النظام يوقف SLA عند انتظار العميل ويستأنفه عند عودة العمل للفريق.
5. الملفات والتعليقات تفصل بين الداخلي والخارجي.
6. كل قرار مؤثر يسجل في Audit Log.
