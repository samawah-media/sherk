# Deliverable Workspace UX: شريك

**المرحلة:** Phase 05 - Information Architecture, UX Model & Role-Based User Flows  
**نوع الوثيقة:** Deliverable Workspace UX  
**الحالة:** Draft for owner review  
**آخر تحديث:** 2026-06-23  

## 1. الغرض

تحديد تجربة مساحة المخرج باعتبارها قلب شريك. المخرج ليس مهمة ولا Ticket، وDrawer المخرج هو المكان الذي يجمع السياق بدون كشف الداخلي للعميل.

## 2. نمط العرض

| النمط | القرار |
| --- | --- |
| Drawer | النمط الافتراضي للتفاصيل من القوائم والKanban. |
| Full page | يستخدم عند Deep Link أو شاشة جوال صغيرة إذا كان Drawer مرهقا. |
| Tabs | تفاصيل، ملفات ونسخ، تعليقات، مراجعة/اعتماد، نشاط. |
| Role-aware content | نفس المخرج يعرض محتوى مختلفا حسب الدور. |

## 3. Header

| العنصر | العميل | الفريق | الإدارة |
| --- | --- | --- | --- |
| اسم المخرج | نعم | نعم | نعم |
| النوع | نعم | نعم | نعم |
| الحالة | مبسطة | كاملة | كاملة |
| SLA | مبسط | عملي | تفصيلي مع delay owner |
| Owner | اختياري مبسط | واضح | واضح |
| النسخة | النسخة المرسلة/النهائية | كل النسخ المصرح بها | كل النسخ المصرح بها |

## 4. Tabs حسب الدور

| Tab | Client | Team | Management |
| --- | --- | --- | --- |
| التفاصيل والمعاينة | نعم، نسخة مرسلة فقط | نعم | نعم |
| المهام | لا | نعم | نعم |
| الملفات والنسخ | ملفات مسموحة فقط | داخلي ومرئي حسب الرؤية | كامل حسب الصلاحية |
| التعليقات | تعليقات العميل فقط | داخلي وعميل حسب scope | داخلي وعميل |
| الجودة | لا | قراءة/تحديث حسب صلاحية | نعم |
| الاعتمادات | سجل خارجي مبسط | حالة داخلية وعميل | كامل |
| النشاط | سجل خارجي مبسط | حسب scope | Audit/Activity كامل |

## 5. Primary Actions

| حالة المخرج | العميل | الفريق | الإدارة/AM |
| --- | --- | --- | --- |
| not_started | لا يظهر غالبا | بدء العمل | تعيين/تعديل |
| in_progress | لا يظهر | رفع/تعليق/طلب مراجعة | متابعة |
| ready_for_internal_review | لا يظهر | انتظار قرار | تعميد أو طلب تعديل |
| internal_changes_requested | لا يظهر | معالجة التعديل | متابعة |
| internally_approved | لا يظهر إلا إذا ملف مسموح | لا إرسال | إرسال للعميل أو جاهز للتسليم |
| waiting_client_approval | اعتماد/طلب تعديل | قراءة ملاحظات إن صرح | تذكير/متابعة |
| client_changes_requested | يرى أن الطلب وصل | معالجة ملاحظات | متابعة |
| client_approved | يرى أنه معتمد | تجهيز | تسليم |
| delivered | يرى النهائي وطلب متابعة | لا تعديل إلا Reopen | إعادة فتح/أرشفة |

## 6. Version Awareness

| المفهوم | UX |
| --- | --- |
| النسخة الداخلية | شارة "داخلية" ولا تظهر للعميل. |
| النسخة المعتمدة داخليا | شارة "معتمدة داخليا" وتؤهل للإرسال. |
| النسخة المرسلة للعميل | تظهر للعميل كنسخة القرار. |
| النسخة النهائية | تظهر في الملفات والتسليم. |
| نسخة جديدة بعد التعميد | تظهر تحذيرا: تحتاج مراجعة داخلية جديدة قبل الإرسال. |

## 7. Approval UX

| القرار | UX |
| --- | --- |
| التعميد الداخلي | شاشة/منطقة تؤكد النسخة، Checklist، وتعليق اختياري/سبب. |
| طلب تعديل داخلي | تعليق داخلي مطلوب. |
| إرسال للعميل | مراجعة المستلمين والنسخة والرسالة. |
| اعتماد العميل | تأكيد مختصر يذكر النسخة. |
| طلب تعديل العميل | تعليق حر مطلوب. |
| Bulk Approval | خارج Drawer، لكن يمكن فتح أي عنصر للفحص الفردي. |

## 8. SLA UX داخل المخرج

| الدور | العرض |
| --- | --- |
| العميل | "بانتظار موافقتك" أو "قيد عمل الفريق" أو "تم التسليم". |
| الفريق | حالة SLA، الموعد، هل ينتظر العميل أو عاد للفريق. |
| الإدارة | Timeline مختصر، delay owner، آخر pause/resume، التصعيد. |

## 9. Safety Rules

| القاعدة | UX |
| --- | --- |
| لا إرسال قبل التعميد | زر الإرسال Disabled مع سبب. |
| لا تسليم قبل اعتماد العميل إذا مطلوب | يظهر مانع واضح. |
| لا تحويل Internal File للعميل بصمت | Dialog حساس مع ملخص الرؤية. |
| لا اعتماد على نسخة Superseded | رسالة "هذه النسخة تم تحديثها، راجع النسخة الحالية". |
| لا تعليق داخلي للعميل | لا يظهر حتى في العدادات الخارجية. |

## 10. Empty and Error States

| الحالة | الرسالة |
| --- | --- |
| لا ملفات | "ما فيه ملفات مرتبطة بهذا المخرج حتى الآن." |
| لا تعليقات | "ما فيه ملاحظات مسجلة على هذا المخرج." |
| لا صلاحية | "ما عندك صلاحية لهذا الإجراء." |
| نسخة Superseded | "تم تحديث النسخة. راجع النسخة الحالية قبل القرار." |
| حفظ فشل | "ما قدرنا نحفظ التغيير. جرّب مرة ثانية." |
