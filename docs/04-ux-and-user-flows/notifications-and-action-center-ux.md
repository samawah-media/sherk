# Notifications and Action Center UX: شريك

**المرحلة:** Phase 05 - Information Architecture, UX Model & Role-Based User Flows  
**نوع الوثيقة:** Notifications and Action Center UX  
**الحالة:** Draft for owner review  
**آخر تحديث:** 2026-06-23  

## 1. الغرض

تصميم الإشعارات داخل التطبيق ومركز الإجراءات بدون إدخال تكاملات خارجية مثل واتساب في V1.

## 2. مبادئ الإشعارات

- الإشعار يفتح سياقا قابلا للتنفيذ.
- لا يرسل إشعار لا يقدر المستخدم يتصرف عليه أو يفهمه.
- لا يكشف الإشعار الداخلي للعميل.
- الجوال مهم جدا لكل الأدوار، خصوصا العميل والفريق.

## 3. أنواع الإشعارات

| النوع | العميل | الفريق | الإدارة |
| --- | --- | --- | --- |
| مخرج ينتظر الموافقة | نعم | لا | نعم كمتابعة |
| تقرير جديد | نعم | لا | نعم |
| طلب تعديل من العميل | لا | نعم | نعم |
| مراجعة داخلية مطلوبة | لا | لا/حسب الدور | نعم |
| تعديل داخلي مطلوب | لا | نعم | نعم |
| SLA at risk/overdue | مبسط عند الحاجة | نعم | نعم |
| دعوة مستخدم | نعم | نعم | نعم |
| ملف رفع/تحويل رؤية | حسب الرؤية | نعم | نعم |

## 4. Action Center

| القسم | محتوى |
| --- | --- |
| يتطلب إجراء مني | موافقات، مراجعات، تعليقات تحتاج رد. |
| تنبيهات SLA | الخطر والتأخير والانتظار. |
| ملفات وتقارير | تقرير جديد، ملف نهائي. |
| الدعوات والصلاحيات | دعوة، انتهاء دعوة، تفويض. |
| النشاط الأخير | ملخص غير مزدحم. |

## 5. إشعار تقرير جديد

| العنصر | التفاصيل |
| --- | --- |
| Trigger | رفع report_file/client_visible أو Mark Client Visible لتقرير. |
| Recipient | Client roles ضمن العميل. |
| Copy | "تمت إضافة تقرير جديد في ملفاتك." |
| Destination | تفاصيل الملف أو ملفات > التقارير. |
| Visibility | لا يذكر تعليقات داخلية أو اسم ملف داخلي. |

## 6. Notification Copy

| الحدث | النص |
| --- | --- |
| مخرج ينتظر العميل | "فيه مخرج جاهز ينتظر موافقتك." |
| تعديل عميل للفريق | "العميل أرسل ملاحظة تعديل على مخرج." |
| SLA خطر | "المخرج قريب من موعده ويحتاج متابعة." |
| SLA بانتظار العميل | "المخرج متوقف بانتظار موافقة العميل." |
| دعوة انتهت | "انتهت صلاحية الدعوة، أعد إرسالها إذا ما زالت مطلوبة." |

## 7. Noise Control

| القاعدة | UX |
| --- | --- |
| تجميع الأحداث المتكررة | "3 تعليقات جديدة على مخرج". |
| أولوية | عاجل، يحتاج إجراء، معلومة. |
| Mark as read | موجود. |
| Snooze | لاحق، ليس ضروريا في V1. |
| إعدادات | Client Admin/user يضبط إشعاراته داخل التطبيق. |

## 8. Out of Scope

| العنصر | السبب |
| --- | --- |
| WhatsApp automation | مؤجل خارج V1. |
| Slack integration | خارج V1. |
| Push native mobile | لا يوجد تطبيق Native في V1. |
| Email strategy التفصيلي | يمكن لاحقا، لا يحسم في UX Phase. |
