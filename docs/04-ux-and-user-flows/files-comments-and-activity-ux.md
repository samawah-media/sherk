# Files, Comments, and Activity UX: شريك

**المرحلة:** Phase 05 - Information Architecture, UX Model & Role-Based User Flows  
**نوع الوثيقة:** Files, Comments, and Activity UX  
**الحالة:** Draft for owner review  
**آخر تحديث:** 2026-06-23  

## 1. الغرض

تصميم تجربة الملفات والتعليقات وسجل النشاط مع احترام visibility والنسخ وAudit.

## 2. ملفات العميل

| النوع | يظهر للعميل؟ | ملاحظات |
| --- | --- | --- |
| internal_only | لا | لا يظهر ولا يعد في الواجهة. |
| client_visible | نعم | بعد سماح واضح. |
| client_uploaded | نعم | ملف أرسله العميل. |
| final_delivery | نعم | بعد التسليم أو الاعتماد. |
| contract_file | نعم حسب الصلاحية | نسخة مبسطة أو ملف عقد. |
| report_file | نعم | داخل الملفات، مع إشعار عند الإضافة. |
| brand_asset | نعم حسب الرؤية | أصول العميل أو الهوية. |

## 3. File UX

| الوظيفة | UX |
| --- | --- |
| معاينة | صور/PDF/فيديو/مستندات عند الإمكان. |
| Fallback | إذا لم تدعم المعاينة: تنزيل أو فتح خارجي. |
| تنزيل متعدد | خارج V1 حسب قرار المالك. |
| تحويل للعميل | إجراء حساس: مراجعة visibility، النسخة، السياق. |
| تحديد Final | منفصل عن الرفع. |

## 4. تقارير العميل

| القرار | UX |
| --- | --- |
| التقارير داخل الملفات | تصنيف "التقارير". |
| يضيفها فريق العمل | رفع كملف report_file/client_visible. |
| إشعار العميل | "تمت إضافة تقرير جديد في ملفاتك." |
| لا Tab مستقل | Confirmed V1. |

## 5. Comments UX

| النوع | من يكتبه | من يراه | UX |
| --- | --- | --- | --- |
| internal_comment | سماوة | سماوة فقط | شارة قفل واضحة. |
| client_comment | العميل أو رد موجه له | العميل وسماوة المصرح لهم | شارة عميل. |
| approval_comment | مع قرار | حسب القرار | مرتبط بالنسخة. |
| system_comment | النظام | حسب السياق | نص مفهوم غير تقني. |

## 6. تعليق حر

بناء على قرار المالك:

- طلب تعديل العميل يعتمد على تعليق حر.
- لا نلزم العميل بقائمة أسباب في V1.
- يمكن إضافة أمثلة نصية خفيفة placeholder فقط.

## 7. Activity Feed

| الفلتر | يدخل V1؟ |
| --- | --- |
| حسب العميل | نعم |
| حسب المخرج | نعم |
| حسب المستخدم | نعم |
| حسب نوع الحدث | نعم |
| حسب ملف/تعليق/اعتماد | نعم |
| حسب التاريخ | نعم |
| internal/external | نعم للإدارة |

## 8. Activity Visibility

| الدور | يرى |
| --- | --- |
| العميل | سجل خارجي مبسط: أرسل، اعتمد، طلب تعديل، ملف أضيف. |
| الفريق | النشاط المرتبط بمخرجاته/عملائه. |
| الإدارة | نشاط كامل ضمن النطاق. |
| Platform | لا content activity روتيني إلا بسياسة دعم. |

## 9. Safety

| الخطر | الحماية UX |
| --- | --- |
| تعليق داخلي ينشر للعميل | التعليق الداخلي لا يتحول؛ يكتب تعليق جديد للعميل. |
| ملف داخلي يتحول بالخطأ | تأكيد حساس وسبب. |
| اعتماد على ملف خاطئ | إظهار نسخة القرار. |
| ضياع مدخلات عند الخطأ | حفظ المسودة عند فشل الحفظ. |
