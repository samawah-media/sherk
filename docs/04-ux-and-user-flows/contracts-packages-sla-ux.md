# Contracts, Packages, and SLA UX: شريك

**المرحلة:** Phase 05 - Information Architecture, UX Model & Role-Based User Flows  
**نوع الوثيقة:** Contracts, Packages, and SLA UX  
**الحالة:** Draft for owner review  
**آخر تحديث:** 2026-06-23  

## 1. الغرض

تحديد تجربة العقود والباقات وSLA للعميل والفريق والإدارة بشكل مفهوم، مع الحفاظ على أن الرصيد مشتق من Ledger وأن SLA مبسط للعميل.

## 2. عقد وباقة العميل

| العنصر | يظهر للعميل | يظهر للإدارة |
| --- | --- | --- |
| المتفق عليه | نعم، كميات وأنواع | نعم، مع تفاصيل الباقة |
| المنجز | نعم | نعم |
| المتبقي | نعم | نعم |
| المحجوز | لا أو مبسط | نعم |
| Ledger كامل | لا | حسب الصلاحية |
| مالية حساسة | لا في V1 | للإدارة حسب الصلاحية |

## 3. Client Contract View

مكونات الشاشة:

- ملخص: "المتفق عليه / تم تسليمه / بانتظار موافقتك / المتبقي".
- بطاقات حسب النوع: منشورات، Reels، تقارير، خطة محتوى.
- روابط للملفات النهائية والتقارير.
- Timeline مبسط للعقد.

## 4. Management Package View

- Commitments وPackage Lines.
- Reserved / Consumed / Remaining.
- مخرجات خارج الباقة.
- انتهاء عقد أو باقة.
- إجراءات حساسة: تعديل رصيد، تجاوز، إلغاء، إعادة حجز.

## 5. SLA للعميل

| الحالة الداخلية | نص العميل المقترح |
| --- | --- |
| on_track | "قيد العمل حسب الخطة" |
| at_risk | "نراجع الجدول ونتابع التسليم" |
| overdue على سماوة | "قيد المتابعة من فريق سماوة" |
| paused_waiting_client | "بانتظار موافقتك" |
| completed | "تم التسليم" |
| cancelled | "تم إيقاف المخرج" |

لا يظهر للعميل:

- أسماء المتسببين داخليا.
- Timeline Segments التفصيلية.
- قرارات SLA الداخلية.

## 6. SLA للإدارة

| العنصر | الغرض |
| --- | --- |
| Status | on_track / at_risk / overdue / paused. |
| Delay Owner | Client / Team / Internal Decision. |
| Pause Reason | سبب التوقف. |
| Time Waiting Client | تقرير عادل. |
| Escalation | تنبيه وتدخل. |
| History | pause/resume/override. |

## 7. SLA للفريق

- يظهر الموعد الداخلي والنهائي.
- يظهر هل المخرج عاد من العميل للتعديل.
- يظهر تحذير عند at-risk.
- لا يسمح بتعديل SLA للفريق العادي.

## 8. Package Exhausted UX

| الحالة | الرسالة |
| --- | --- |
| لا يوجد رصيد كاف | "الرصيد غير كاف لهذا النوع من المخرجات." |
| مخرج إضافي | "يحتاج موافقة إدارية كمخرج خارج الباقة." |
| عميل يرى المتبقي | "المتبقي من باقتك: 0 Reels." |

## 9. Reopening UX

إعادة الفتح بعد التسليم:

- تظهر للإدارة فقط كإجراء حساس.
- العميل يستخدم "طلب متابعة".
- لا يغير الرصيد ولا SLA تلقائيا.
- يجب تحديد السبب.

## 10. Open Questions

| السؤال | توصية UX |
| --- | --- |
| أيام عمل أم تقويمية؟ | تعرض كسياسة Tenant لاحقا، ولا تفصل للعميل الآن. |
| عتبة at-risk | تستخدم داخليا وتوثق لاحقا. |
| Carry Forward | لا يظهر للعميل إلا عند اعتماده. |
