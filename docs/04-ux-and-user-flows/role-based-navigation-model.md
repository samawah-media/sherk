# Role-Based Navigation Model: شريك

**المرحلة:** Phase 05 - Information Architecture, UX Model & Role-Based User Flows  
**نوع الوثيقة:** Role-Based Navigation Model  
**الحالة:** Draft for owner review  
**آخر تحديث:** 2026-06-23  

## 1. الغرض

تحديد نموذج التنقل لكل دور بدون تحويل Sidebar إلى قائمة بكل خصائص النظام.

## 2. بوابة العميل

التابات الخمسة المعتمدة:

| Tab | يظهر لمن | السبب | Primary Action |
| --- | --- | --- | --- |
| الرئيسية | كل أدوار العميل | ملخص سريع لما تم وما ينتظر العميل | فتح الموافقات أو الملفات |
| بانتظار موافقتي | Approver، Reviewer/View-only بشكل محدود | قرار العميل الأهم | اعتماد أو طلب تعديل |
| العقد والمتابعة | كل أدوار العميل | المتبقي والمنجز والتقدم | فتح تفاصيل مخرج/ملف |
| الملفات | كل أدوار العميل | التقارير والملفات النهائية | معاينة أو تنزيل |
| الإعدادات | Client Admin والذات | المستخدمون والإشعارات | إدارة/طلب دعوات |

### Bulk Approval

يدخل في "بانتظار موافقتي" فقط عندما:

- كل العناصر المحددة من نفس العميل.
- كل العناصر مرسلة للاعتماد.
- المستخدم `client_approver`.
- تظهر معاينة ملخصة وعدد النسخ قبل التأكيد.

| القرار | التصنيف |
| --- | --- |
| Bulk Approval مسموح في V1 إذا كان كتحديد متعدد واضح وآمن. | Confirmed |

## 3. مساحة الفريق

| Nav Item | يدخل V1؟ | السبب | ملاحظات |
| --- | --- | --- | --- |
| مهامي | نعم | أسرع دخول يومي للفريق | مهم على الجوال. |
| لوحة العمل | نعم | تشغيل Kanban الداخلي | Desktop-first مع بديل قائمة. |
| المخرجات | نعم | بحث وفلاتر وبديل Accessible | ليست تكرارا للوحة؛ هي table/list. |
| الملفات | نعم | أصول العمل ومرفوعات العميل | رؤية حسب scope. |
| الإشعارات | نعم | توجيهات العميل وSLA والمراجعات | Push داخل التطبيق لاحقا. |
| البحث العام | نعم | مطلوب من المالك | كحقل أعلى لا Tab مستقل غالبا. |
| إعدادات | لا للفريق العادي | لا يحتاجها يوميا | تظهر للمديرين فقط. |

## 4. لوحة الإدارة

| Nav Item | يدخل V1؟ | السبب |
| --- | --- | --- |
| نظرة عامة | نعم | متابعة الاختناقات السريعة. |
| العملاء | نعم | إدارة 4 عملاء كبداية والتوسع لاحقا. |
| المخرجات | نعم | قلب العمل الإداري. |
| لوحات العمل | نعم | Kanban عابر أو حسب عميل. |
| SLA والجودة | نعم | قيمة أساسية في V1. |
| الفريق | نعم | ضغط العمل وإعادة الإسناد. |
| العقود والباقات | نعم | المتبقي والحجز والاستهلاك. |
| الملفات | نعم | تحويل الرؤية والتقارير والنهائيات. |
| التقارير | نعم للإدارة | تقارير تشغيلية داخلية، لا تقارير العميل. |
| الإعدادات | نعم | أدوار، دعوات، Tenant basics. |
| البحث العام | نعم | مطلوب، حقل علوي دائم. |

## 5. إدارة المنصة

| Nav Item | يدخل V1؟ | ملاحظات |
| --- | --- | --- |
| Tenants | نعم كحوكمة محدودة | سماوة أول Tenant. |
| Platform Users | نعم محدود | مالك/إدارة المنصة. |
| Support Sessions | Assumed | مقيد جدا، ليس content superuser. |
| Billing SaaS | لا | خارج V1. |
| White Label | لا | مؤجل حسب قرار المالك. |

## 6. قواعد إظهار العناصر

| القاعدة | التصنيف |
| --- | --- |
| لا يظهر زر لا يستطيع المستخدم استخدامه إلا إذا كان Disabled مع سبب تعليمي مختصر. | Assumed |
| العميل لا يرى Kanban أو جودة داخلية أو Audit داخلي. | Confirmed |
| الفريق يرى الإدارة فقط إذا يحمل دورا إداريا. | Confirmed |
| مدير الحساب يرى عملاءه ومخرجاتهم، ويرسل للعميل بعد التعميد إذا خول. | Confirmed |
| Global Search لا يرجع نتائج خارج scope ولو كتلميح. | Confirmed |

## 7. Mobile Navigation

| التجربة | نمط الجوال |
| --- | --- |
| العميل | Bottom navigation للتابات الخمسة أو قائمة مختصرة، مع CTA ثابت للموافقة. |
| الفريق | Bottom tabs: مهامي، إشعارات، بحث، المزيد. Kanban يتحول لقائمة مراحل. |
| الإدارة | Mobile يدعم القراءة والتدخلات المهمة، وDesktop يبقى الأساسي للوحة الشاملة. |
