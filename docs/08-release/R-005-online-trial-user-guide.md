# دليل تجربة Sharik Online Trial

## 1. رابط البيئة

- رابط Vercel الحالي للتجربة: https://sharik-platform.vercel.app
- اسم بيئة staging المعتمد لـ R-005: `sharik-internal-trial-staging`
- ملاحظة: الرابط الحالي للتجربة الداخلية فقط، ولا يعني اعتماد Production أو استخدام بيانات حقيقية.

## 2. حسابات التجربة

> لا تكتب كلمات المرور داخل GitHub. سيتم تسليم كلمة المرور خارج GitHub.

| الدور | البريد التجريبي | كلمة المرور |
|---|---|---|
| Admin | `tenant-admin@r005.example.test` | سيتم تسليم كلمة المرور خارج GitHub |
| Account Manager | `account-manager@r005.example.test` | سيتم تسليم كلمة المرور خارج GitHub |
| Client Viewer | `client-viewer@r005.example.test` | سيتم تسليم كلمة المرور خارج GitHub |

كل البيانات في هذه التجربة اصطناعية ومخصصة لـ R-005 فقط.

## 3. سيناريو تجربة Admin

1. افتح رابط البيئة وسجل الدخول بحساب Admin.
2. افتح صفحة العملاء.
3. افتح عميلًا تجريبيًا مثل `R-005 Client Alpha`.
4. راجع العقد/الباقة من روابط العميل أو من الملخص التجاري.
5. افتح صفحة المخرجات.
6. افتح `لوحة Kanban الداخلية`.
7. غيّر حالة مخرج مسموح من قائمة الحالة، ثم اضغط `تحديث الحالة`.
8. راجع أن نسبة التقدم على بطاقة المخرج تغيرت بعد تحديث الحالة.

## 4. سيناريو تجربة Account Manager

1. سجل الدخول بحساب Account Manager.
2. تأكد أنه يرى العميل المصرح له فقط، والمتوقع في التجربة هو `R-005 Client Alpha`.
3. افتح لوحة Kanban الخاصة بالعميل المصرح.
4. حدّث حالة مخرج مسموح، وتأكد من ظهور رسالة نجاح.
5. جرّب انتقال حالة غير مسموح من القائمة أو عبر مسار غير مصرح، وتأكد أن النظام يرفضه برسالة آمنة أو يمنع الاختيار.
6. تأكد أنه لا يرى عميلًا آخر مثل `R-005 Client Beta`.

## 5. سيناريو تجربة Client Viewer

1. سجل الدخول بحساب Client Viewer.
2. تأكد أنه يرى واجهة العميل فقط.
3. تأكد أنه لا يرى `لوحة Kanban الداخلية`.
4. تأكد أنه لا يستطيع فتح روابط board الداخلية.
5. تأكد أنه لا تظهر له تعليقات أو بيانات داخلية أو تفاصيل تشغيل خاصة بفريق سماوة.

## 6. ما هو خارج التجربة الآن

- الملفات
- التعليقات
- التعميد الكامل
- اعتماد العميل الكامل
- Drag/drop
- Billing
- Social scheduling
- AI

## 7. ملاحظات مطلوبة من الفريق

- هل الدخول للحسابات سهل؟
- هل العملاء واضحون؟
- هل المخرجات واضحة؟
- هل الـ board واضح؟
- هل تغيير الحالة مفهوم؟
- هل الواجهة العربية مناسبة؟
- هل توجد أخطاء أو بطء أو لبس؟
